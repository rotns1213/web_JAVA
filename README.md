# 스프링 웹 MVC 프로젝트의 구조 (CH02)

## 스프링 웹 MVC?
* 스프링이 제공하는 웹 애플리케이션 개발 전용 프레임워크
  * M : model 모델 => 애플리케이션의 데이터가 들어 있는 객체
  * V : view 뷰 => 애플리케이션의 비지니스 로직을 포함하는 자바 클래스 (@Controller)
  * C : controller 컨트롤러 => 모델의 정보(data)를 특정 형식에 맞게 표현 (JSP, Thymeleaf, Freemarker 등)
  
* 웹 애플리케이션의 MVC 사이의 의존 관계 및 처리 흐름을 스프링 컨테이너가 관리함
* 프런트 컨트롤러는 애플리케이션의 흐름을 관리함 (Spring Framwork의 DispatcherServlet)

## MVC의 기본 구성 요소
* 디스패처 서블릿 : 클라이언트로 부터 요청 전달 받음 -> 컨트롤러에 전달 -> 컨트롤러가 반환한 결과 값 뷰에 전달 -> 사용자에게 보여줄 응답을 생성하는 등의 모든 흐름을 담당
* 햄들러 매핑 (Handler Mapping) : 클라이언트의 요청 URL에 대해 어떤 컨트롤러가 처리할지를 결정
* 핸들러 어댑터 (Handler Adapter) : 매핑 클래스에 의해 결정된 컨트롤러 호출
* 컨트롤러 : 클라이언트의 요청 처리 후 결과 반환 -> 응답 결과에서 보여줄 데이터를 모델에 담아서 뷰로 전달 (Controller - model(data) - view)
* 모델 & 뷰 : 컨트롤러가 처리한 결과 정보 및 뷰 선택에 필요한 정보를 담는 객체
* 뷰 리졸버 (View Resolver) : 컨트롤러의 처리 결과를 사용자에게 보여줄 뷰를 결정
* 뷰 : 컨트롤러의 처리 결과 화면을 생성, JSP를 비롯한 다양한 뷰 템플릿 엔진이 사용됨. 클라이언트에 요청 처리 결과 전송함
